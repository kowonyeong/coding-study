# 관리자 운영 가이드

이 문서는 코딩 테스트 스터디 관리자가 저장소를 안정적으로 운영하기 위한 상세 가이드입니다.

## 1. 운영 목표

- 스터디원이 절차에 신경 쓰지 않고 풀이에 집중하도록 돕기
- 기록(문제, 풀이, 상태)을 한 저장소에서 일관되게 관리하기
- 주차별 진행률과 참여도를 쉽게 파악하기

## 2. 기본 운영 방식

- 브랜치 전략 없이 `main`에 바로 push 하는 간편 운영
- 언어별 구역 분리를 통해 충돌 최소화
- 주차별 문제표(`problems/weekXX/README.md`)를 기준으로 진행상황 관리

## 3. 초기 세팅 체크리스트

- [ ] `solutions/` 하위 언어 폴더 확인
- [ ] 주차 폴더(`problems/week01`, `problems/week02`...) 준비
- [ ] README 최신화 여부 확인
- [ ] 운영 규칙(`docs/STUDY_RULES.md`) 공지

## 4. 주간 운영 루틴 (권장)

### 주 시작 전
1. 이번 주 주차 파일 생성(예: `problems/week03/README.md`)
2. 문제 후보 3~5개 등록
3. 담당자 배정

### 주중
1. 진행중/완료 상태 업데이트 요청
2. 미제출 인원 리마인드
3. 난이도 쏠림(너무 쉬움/어려움) 점검

### 주말/마감
1. 완료된 풀이 링크 점검
2. 누락 파일/오타/경로 오류 확인
3. 다음 주차 파일 초안 준비

## 5. 언어별 구역 운영 방법

권장 폴더 구조:
- `solutions/python/<이름>/`
- `solutions/java/<이름>/`
- `solutions/cpp/<이름>/`
- `solutions/javascript/<이름>/`

운영 팁:
- 스터디원 등록 시 언어를 먼저 확정
- 언어 변경 시 기존 폴더는 보관하고 새 언어 경로 추가
- 동일 문제를 다중 언어로 푸는 경우 언어별로 각각 업로드

## 6. 품질 관리 기준

관리자가 최소한으로 확인할 항목:
- 파일 경로가 맞는가?
- 파일명이 규칙에 맞는가?
- 주차 문제표의 상태가 최신인가?
- 문제 링크가 유효한가?

## 7. 자주 발생하는 이슈와 대응

### Q1) 같은 파일 충돌이 자주 납니다.
- 원인: 공통 파일 동시 수정
- 대응: 주차 문제표 업데이트 시간을 분리하거나 담당자를 정해 순차 반영

### Q2) 업로드 경로가 자주 틀립니다.
- 대응: README의 언어별 경로 예시를 고정하고, 첫 주에는 관리자 1회 검수

### Q3) 커밋 메시지가 제각각입니다.
- 대응: 공지에 메시지 예시를 고정 문구로 제공

## 8. 추천 공지 템플릿

```text
[이번 주 스터디 안내]
- 주차: week03
- 문제 등록 마감: 수요일 23:59
- 풀이 업로드 마감: 일요일 23:59
- 업로드 경로: solutions/<language>/<name>/
- 작업 방식: main 브랜치로 바로 push
```

## 9. 관리자 인수인계 체크포인트

- 현재 주차 진행률
- 미제출자 명단
- 다음 주 문제 후보
- 폴더/파일 네이밍 규칙
- 공지 채널(디스코드/카톡 등) 운영 방식
