# 스터디 운영 규칙 (간편 운영 버전)

## 1) 운영 원칙
- 브랜치 전략 없이 `main`에 바로 반영합니다.
- 목표는 "협업 절차 최소화 + 풀이 집중"입니다.

## 2) 언어별 구역 분리 원칙
- 언어별 기본 구역:
  - `solutions/python/`
  - `solutions/java/`
  - `solutions/cpp/`
  - `solutions/javascript/`
- 스터디원은 본인 언어 구역 중심으로 작업합니다.

## 3) 문제 관리 원칙
- 주차별 문제는 `problems/weekXX/README.md`에서 관리합니다.
- 문제 원문 전체 복붙 금지, 링크와 핵심 요약만 기록합니다.

## 4) 커밋/푸시 원칙
- 작업 후 바로 커밋하고 `main`에 push 합니다.
- 권장 커밋 메시지:
  - `feat: week02 완전탐색 풀이 추가 (python/홍길동)`
  - `docs: week02 문제 상태 업데이트`

## 5) 충돌 방지 원칙
- 같은 파일을 여러 명이 동시에 편집하지 않도록 최소한의 소통을 합니다.
- 공통 문서(`README.md`, 주차 문제표) 편집 전에는 채팅으로 먼저 알립니다.
